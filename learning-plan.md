## 学习路径

### 第一阶段：基础补齐（第1-2周）

#### Week 1: AI 基础
**每日投入**: 2小时

**Day 1-2: LLM（大语言模型）**
- 学习内容：LLM 能做什么，不能替代什么
- Handbook 链接：https://aiweb3.school/zh/handbook/ai/llm/
- 实践任务：使用 ChatGPT/Claude 完成一个简单任务
- 打卡要点：理解 LLM 的能力边界

**Day 3-4: Prompt（提示词）**
- 学习内容：如何把任务目标、边界和输出格式写清楚
- Handbook 链接：https://aiweb3.school/zh/handbook/ai/prompt/
- 实践任务：编写 3 个不同场景的 Prompt
- 打卡要点：掌握 Prompt 工程基础

**Day 5-6: Context（上下文）**
- 学习内容：模型一次能看见什么，哪些信息可信
- Handbook 链接：https://aiweb3.school/zh/handbook/ai/context/
- 实践任务：分析一个长文档的上下文处理
- 打卡要点：理解上下文窗口和信息优先级

**Day 7: RAG（检索增强生成）**
- 学习内容：如何把外部知识接入模型
- Handbook 链接：https://aiweb3.school/zh/handbook/ai/rag/
- 实践任务：搭建一个简单的 RAG 系统
- 打卡要点：理解 RAG 的工作原理

#### Week 2: Web3 基础
**每日投入**: 2小时

**Day 8-9: Network（网络）**
- 学习内容：区块、共识、L2、RPC 和链上状态
- Handbook 链接：https://aiweb3.school/zh/handbook/web3/network/
- 实践任务：使用 RPC 查询链上数据
- 打卡要点：理解区块链网络基础

**Day 10-11: Cryptography（密码学）**
- 学习内容：哈希、公私钥、签名
- Handbook 链接：https://aiweb3.school/zh/handbook/web3/cryptography/
- 实践任务：生成一对公私钥并签名
- 打卡要点：掌握密码学基础概念

**Day 12-13: Wallet（钱包）**
- 学习内容：钱包不只是登录按钮，而是身份和签名入口
- Handbook 链接：https://aiweb3.school/zh/handbook/web3/wallet/
- 实践任务：安装 MetaMask 并完成一笔交易
- 打卡要点：理解钱包的工作原理

**Day 14: Smart Contract（智能合约）**
- 学习内容：链上规则如何部署、调用和更新状态
- Handbook 链接：https://aiweb3.school/zh/handbook/web3/smart-contract/
- 实践任务：阅读一个简单的智能合约
- 打卡要点：理解智能合约基础

### 第二阶段：AI × Web3 Bridge（第3-4周）

#### Week 3: Agent 与工具调用
**每日投入**: 2小时

**Day 15-16: Agent（智能体）**
- 学习内容：如何让模型进入工具调用和多步执行

**Day 17-18: MCP（模型上下文协议）**

- Handbook 链接：https://aiweb3.school/zh/handbook/ai/mcp/

**Day 19-20: Web3 Tool Use（Web3 工具使用）**

- Handbook 链接：https://aiweb3.school/zh/handbook/bridge/web3-tool-use/
- 实践任务：让 Agent 查询链上余额
- 打卡要点：理解 Agent 如何调用 Web3 工具

**Day 21: Agent Workflow（Agent 工作流）**
- 学习内容：哪些步骤适合自动化，哪些必须 human-in-the-loop

**Day 26-27: Settlement & Escrow（结算与托管）**

### 晚上（必做，2小时）
1. **学习（60分钟）**

   - 观看相关视频/教程

2. **实践（45分钟）**
   - 完成实践任务
   - 编写代码/实验
   - 记录问题

3. **总结（15分钟）**
   - 填写每日学习笔记
   - 生成打卡草稿
   - 整理问题反馈

## Handbook Feedback 流程

### 反馈类型
1. **内容问题**：概念不清楚、表述不准确
2. **技术问题**：代码错误、链接失效
3. **结构建议**：章节组织、内容顺序
4. **补充建议**：缺少的案例、代码示例

### 反馈格式
```
反馈类型：[类型]
Handbook 链接：[页面链接]
问题描述：[具体问题]
建议改法：[建议方案]
来源：[反馈来源]
```

### 反馈提交
1. 在 `handbook-feedback/` 目录创建文件
2. 文件名格式：`YYYY-MM-DD-feedback-XX.md`
3. 填写反馈内容
4. Commit and Push
