# Handbook Feedback 模板

## 反馈格式

每条反馈请按以下格式填写：

```
反馈类型：[内容问题/技术问题/结构建议/补充建议]
Handbook 链接：[页面链接]
问题描述：[具体问题描述]
建议改法：[建议的修改方案]
来源：[反馈来源，如学习过程、实践发现等]
```

## 示例

### 示例 1：内容问题
```
反馈类型：内容问题
Handbook 链接：https://aiweb3.school/zh/handbook/ai/llm/
问题描述：LLM 章节中关于"上下文窗口"的解释不够详细，新手可能难以理解
建议改法：增加一个具体的例子，说明上下文窗口大小如何影响模型能力
来源：学习过程中的困惑
```

### 示例 2：技术问题
```
反馈类型：技术问题
Handbook 链接：https://aiweb3.school/zh/handbook/web3/smart-contract/
问题描述：智能合约示例代码中的 Remix 链接已失效
建议改法：更新为有效的 Remix 链接或提供备用链接
来源：实践任务中发现
```

### 示例 3：结构建议
```
反馈类型：结构建议
Handbook 链接：https://aiweb3.school/zh/handbook/bridge/
问题描述：AI × Web3 Bridge 章节内容较多，建议增加一个概览页面
建议改法：在 Bridge 章节开头增加一个概览页面，介绍各子章节的关系
来源：学习过程中的建议
```

### 示例 4：补充建议
```
反馈类型：补充建议
Handbook 链接：https://aiweb3.school/zh/handbook/ai/agent/
问题描述：Agent 章节缺少实际案例，建议增加一个完整的 Agent 开发案例
建议改法：增加一个从需求分析到代码实现的完整案例
来源：学习过程中的需求
```

## 反馈提交流程

1. **发现问题**：在学习过程中发现问题或建议
2. **记录反馈**：按照模板格式记录反馈
3. **创建文件**：在 `handbook-feedback/` 目录创建文件
4. **命名规则**：`YYYY-MM-DD-feedback-XX.md`（XX 为序号）
5. **Commit and Push**：提交到 GitHub 仓库

## 注意事项

- 反馈要具体、明确
- 提供改进建议
- 注明反馈来源
- 保持礼貌和建设性
- 避免重复反馈
