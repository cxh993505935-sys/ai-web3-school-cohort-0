## 📝 学习笔记

### 一、LLM 大语言模型

**核心概念：**
- LLM = 经过海量文本训练的 AI 模型，学会了语言规律
- "大"体现在：参数量大（数十亿~万亿）、训练数据大（TB~PB 级）、计算资源大
- 核心架构：**Transformer**（2017 Google 提出），核心是**自注意力机制**
- 基本原理：预测"下一个最可能的 Token"来生成文本

**两阶段训练：**
1. **预训练**：海量文本无监督学习，核心任务 = Next Token Prediction
2. **微调/对齐**：高质量问答 + 人类反馈（SFT、RLHF），让模型更安全有用

**LLM 能做什么：**
- 文本生成、对话交流、文本转换（翻译/摘要/改写）
- 代码生成、推理分析、创意内容

**LLM 不能做什么：**
- 🔴 幻觉（Hallucination）— 编造不存在的事实
- 🔴 知识截止 — 不知道训练后发生的事件
- 🔴 无法访问外部世界 — 原生不能上网/执行代码
- 🔴 不擅长精确计算
- 🔴 上下文窗口有限、记忆不持久

**关键术语：**
- **Token**：最小处理单位，中文约 1-2 Token/字
- **Context Window**：一次能处理的最大 Token 数
- **Temperature**：控制输出随机性，高→多样，低→确定
- **Embedding**：文本→数字向量，用于语义搜索
- **RAG**：先检索再生成，减少幻觉

**核心原则：** 把 LLM 当作能力超强但需要明确指令的新同事。

### 二、Prompt 提示工程

**有效 Prompt 四要素：**
1. **指令** — 做什么
2. **上下文** — 背景信息
3. **输入数据** — 待处理内容
4. **输出格式** — 期望结果形态

**常见技巧：**
- **Zero-shot**：零样本直接给任务
- **Few-shot**：提供几个示例让模型学习模式
- **Chain of Thought**：引导模型逐步推理
- **Role Prompting**：指定角色身份回答
- **Prompt Chaining**：复杂任务拆成多步串联

**写作要点：**
- 清晰性：避免歧义，一次一个任务
- 具体性：明确约束（字数、格式、风格）
- 结构化：用分隔符组织内容
- 迭代优化：根据输出持续调整

### 三、Web3 区块链网络

**区块（Block）：**
- 区块链的基本数据单位，包含：区块头、交易列表、哈希、Nonce
- 流程：交易发起 → 广播网络 → 矿工打包进区块 → 添加到链 → 状态更新

**共识机制：**
- **PoW（工作量证明）**：计算竞争记账权，安全但耗能（Bitcoin）
- **PoS（权益证明）**：质押代币获得验证资格，低能耗（Ethereum）

**L2（Layer 2）：**
- 建立在主链之上的扩展方案，继承 L1 安全性
- 速度更快、手续费更低（L1 几十美元 → L2 几美分）
- 代表：Arbitrum、Optimism、zkSync

**RPC（Remote Procedure Call）：**
- 钱包/应用与区块链通信的桥梁，相当于"API 接口"
- 服务商：Infura、Alchemy、QuickNode
- 可以查询余额、发送交易等

**链上状态：**
- 所有账户的当前数据：余额、合约存储、代币、NFT
- 每次交易导致状态转换（如 A 转 2 ETH 给 B → 双方余额变化）

**关键术语：**
- **Gas 费**：执行交易的手续费
- **节点**：保存完整账本的计算机
- **区块哈希**：区块的唯一标识
- **不可篡改性**：改一处会导致后续全部对不上
